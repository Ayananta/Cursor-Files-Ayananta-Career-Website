Information Architecture Service

Transform your content structure with strategic information architecture that enables content reuse, rapid localization, and scalable documentation delivery.

Phase 1: Assessment and Planning (Steps 1-3)

I conduct a content audit to identify exact (identical content), near (minor variations), and thematic duplicates (same concept, different wording). I then estimate the effort involved. You get an Audit Report. I define reuse granularity levels, establish reuse thresholds, and define naming conventions for reusable components. You get a Reuse Strategy doc, containing policies and guidelines. Next, I design the repository. You get a Repository Structure.

  1. Conduct content audit to identify reuse opportunities and estimate effort
    1. Export content inventory from CCMS.
    2. Run TEXTML queries to identify duplicate content.
    3. Categorize duplicates (exact, near, and thematic).
    4. Calculate baseline metrics (total topics, estimated duplication %, and maintenance burden).
  2. Define reuse strategy to establish policies and priorities
    1. Define reuse levels (topic, section, block, phrase).
    2. Create a reuse policy (legal disclaimers, safety warnings, common steps, etc).
    3. Establish reuse thresholds (3+ times, twice, once).
    4. Define naming conventions for reusable components.
  3. Design the repository structure to organize content for reuse
    1. Create dedicated reuse library structure in CCMS (product-specific, version-sp'fic, lang-sp'fic, etc).
    2. Configure folder permissions (based on user role).
    3. Create CCMS collections for logical grouping (legal boilerplate, product descr'ns etc).

Phase 2: Infrastructure Setup (Steps 4-6)

I shall build an initial set of reusable components. You get a Reusable Content Lib. I implement conref mechanism to enable content reference functionality. You get a working Conref Infrastructure with test cases. Next, I implement conkeyref for variable content to enable context-sensitive content reuse. You get a Conkeyref System supporting context-sensitive content reuse.

  1. Create reusable content library to build initial set of reusable components
    1. Extract high-priority reusable content identified in audit.
    2. Create source topics for conref by compiling identif'd reusable con't of similar type in their respective source topics.
    3. Set metadata on reusable topics.
    4. Register topics in CCMS.
  2. Implement Conref mechanism to enable content reference functionality
    1. Configure IxiaSoft conref support.
    2. Create conref reference patterns.
    3. Set up conref push mechanism for cases where content needs to be inserted into specific locations.
    4. Test conref resolution.
  3. Implement conkeyref for variable content to enable context-sensitive reuse
    1. Create key definition maps for reusable content.
    2. Ref key definitions in root maps.
    3. Use conkeyref in topics.
    4. Test conkeyref with different key scopes (build outputs with different root maps, verify correct variants appear in each output).

Phase 3: Content Migration (Steps 7-8)

I implement content reuse. I profile content and use variables. I create guidelines for when to use Direct conref, Filtered conref, and Conkeyref. You get Reuse-Optimized Content.

  1. Execute reusing of content to implement reuse across the content repository
    1. Group reuse candidates by type & prioritize groups for migration.
    2. Execute migration workflow (extract, clean, generalize, assign ID, check-in, replace, validate, review).
    3. Use CCMS to track migration.
    4. Document conref locations in a spreadsheet.
  2. Use profiling and variables for content that requires conditional reuse
    1. Implement filtered conref for variations; for example, content with audience filtering.
    2. Use conref range for multiple elements.
    3. Implement conref with profiling for product variations.
    4. Create guidelines for scenario-based conref usage.
